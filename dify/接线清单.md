# Dify 订单分支接线清单（FastAPI HTTP）

目标：把 Chatflow 订单物流从 Code Mock 换成 HTTP GET，调用自建 FastAPI：

`GET {BACKEND_BASE_URL}/api/v1/tools/query_order?order_id=...`

**DSL 文件**：`智能体客服-后端已接线.yml`  
（Downloads 里的 `智能体客服 (1).yml` 已含 HTTP 节点，内容与已接线版一致，可保留不动。）

---

## 1. 启动后端

```bash
cd backend   # 或你的 FastAPI 目录
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

本地自检：

```bash
curl "http://127.0.0.1:8000/api/v1/tools/query_order?order_id=PP20260713001234"
```

应返回 JSON（含 `ok` / `data` 或错误字段）。超时与重试由后端 tools 门面完成，Dify 侧只做一次 HTTP 调用。

---

## 2. Dify Cloud：用 ngrok 暴露公网

Dify Cloud 无法访问本机 `127.0.0.1`，需先打隧道：

```bash
ngrok http 8000
```

把 ngrok 给出的 `https://xxxx.ngrok-free.app`（**不要**末尾斜杠）记下来，下一步写入 `BACKEND_BASE_URL`。

自托管 Dify 若与后端同网，可直接用 `http://host.docker.internal:8000` 或内网 IP。

---

## 3. 在 Dify 设置环境变量

工作室 → 环境变量，确认：

| 名称 | 说明 | 示例 |
|------|------|------|
| `BACKEND_BASE_URL` | FastAPI 可访问基址，无末尾 `/` | `https://xxxx.ngrok-free.app` 或 `http://127.0.0.1:8000` |

---

## 4. 导入 DSL

1. 工作室 → 创建 / 导入 DSL  
2. 选择 `智能体客服-后端已接线.yml`  
3. 若已有旧应用：可新建「后端已接线」版本，或在旧应用上手动叠加 HTTP 订单分支  

导入后再次确认环境变量 `BACKEND_BASE_URL` 已填写。

---

## 5. 重新挂载知识库与模型

导入后绑定通常会丢，请检查：

1. **知识检索 RAG**：重新挂载 4 个业务知识库  
2. **意图规划 / LLM-订单 / LLM-RAG**：换成当前账号可用模型  
3. 保存草稿后再发布  

---

## 6. 联调话术

| 输入 | 期望 |
|------|------|
| `怎么查我的订单？`（无订单号） | 「请提供订单号」，举例 PP…，给热线 |
| `查询订单 PP20260713001234` | HTTP 成功 → 解析 → LLM 说明真实状态（不提 Mock/HTTP） |
| `查询订单 PP99999999999999` | HTTP 通但业务失败 → LLM 按错误提示致歉转人工 |
| 停掉后端再问有单号的订单 | HTTP fail-branch →「订单接口失败转人工」 |
| `我想投诉，转人工` | 分类器 HITL 固定话术 |
| `三个型号有什么区别？` | 常规 RAG 分支不受影响 |

---

## 7. 导入失败时的手动叠加（UI）

若 DSL 导入报错，可在合格版 Chatflow 上手动改订单分支：

1. 原「工具 query_order(Mock)」Code 改为「抽取订单号」（输出 `order_id` / `need_order_id`）  
2. 新增 IF：`need_order_id is "true"` → Answer「请提供订单号」  
3. false → HTTP Request GET：  
   `{{#env.BACKEND_BASE_URL#}}/api/v1/tools/query_order?order_id={{#抽取.order_id#}}`  
   无鉴权；超时 connect 10 / read 15 / write 10；错误策略用**失败分支**；ngrok 时可关 SSL 校验  
4. HTTP 成功 → Code「解析HTTP工具响应」→ LLM「订单工具答复」→ Answer  
5. HTTP 失败分支 → Answer「订单接口失败转人工」  
6. 环境变量增加 `BACKEND_BASE_URL`；删除旧的 Code→LLM 直连边  

完整边关系见同目录 `README.md` 架构图。
