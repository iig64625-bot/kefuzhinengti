## 架构（后端已接线）

```
用户输入
  → 意图规划(Classifier)
       ├─ 人工干预 → 固定转人工回复（HITL）
       ├─ 订单物流 → Code:抽取订单号 → IF 是否缺单号
       │               ├─ true  → 请提供订单号
       │               └─ false → HTTP GET /api/v1/tools/query_order
       │                            ├─ success → Code:解析响应 → LLM → 回复
       │                            └─ fail-branch → 订单接口失败转人工
       └─ 常规咨询 → 知识检索 → Code:空检索检查 → IF
              ├─ 空 → 兜底转人工话术
              └─ 有结果 → LLM(规划/证据/校验) → 回复
```

环境变量：`BACKEND_BASE_URL`（无末尾斜杠）。超时/重试在后端 tools 门面完成。

## 标准对照

| 视频标准 | 本 YAML 如何体现 |
|---------|-----------------|
| 1 真实业务 | 选购/售后/订单/退换 + 开场白业务说明 |
| 2 后端工程 | HTTP 对接 FastAPI `/api/v1/tools/query_order` |
| 3 Agent 能力 | 规划分流 + 工具调用 + 失败恢复 + 结果验证 |
| 4 上下文工程 | System/RAG Context/Memory/工具JSON 分层注入 |
| 5 可观测 | 引用开关；trace；后端 traces |
| 6 人工干预 | 分类器强制 HITL；空检索兜底；HTTP 失败转人工 |
