# Dify Agent 合格版说明（对齐抖音视频 6 条标准）

文件：`智能体客服 (1).yml` / `starsleep-agent/dify/智能体客服-合格版.yml`

## 架构

```
用户输入
  → 意图规划(Classifier)
       ├─ 人工干预 → 固定转人工回复（HITL）
       ├─ 订单物流 → Code:query_order(Mock) → LLM验证答复 → 回复
       └─ 常规咨询 → 知识检索 → Code:空检索检测 → IF
              ├─ 空 → 兜底转人工话术
              └─ 有结果 → LLM(规划/证据/校验) → 回复
```

## 标准对照

| 视频标准 | 本 YAML 如何体现 |
|---------|-----------------|
| 1 真实业务 | 选购/售后/订单/退款 + 开场白业务说明 |
| 2 后端工程 | 作品集层：Mock 工具可替换真实 OMS；Dify 会话承接日志 |
| 3 Agent 能力 | 规划分流 + 工具调用 + 失败恢复(retry/兜底) + 结果验证 |
| 4 上下文工程 | System/RAG Context/Memory/工具JSON 分层注入 |
| 5 可观测 | 引用开关；trace=retrieval_empty/hit；回答标注证据来源 |
| 6 人工干预 | 分类器强制 HITL；空检索兜底；投诉/安全场景 |

## 导入后检查清单

1. 重新挂载 4 个知识库到「知识检索-RAG」
2. 确认模型 gpt-5.4（或换成你可用模型）在分类器/两个 LLM 节点可用
3. 发布后测试：
   - `三个型号有什么区别？` → 常规 RAG
   - `查询订单 PP20260713001234` → Mock 工具已发货
   - `我想投诉，转人工` → HITL 固定话术
   - 故意问无关冷门问题 → 空检索兜底（视知识库情况）
