# 工作流架构详解

## 整体流程

本工作流采用 **分类路由 + RAG** 架构，核心思路：

1. 用户提问先经过 **问题分类器**，自动识别问题类型
2. 根据分类结果，路由到对应的知识检索节点
3. 检索结果送入专属 LLM，生成针对性回答
4. 回答直接返回给用户

## 节点详解

### 1. 开始节点 (Start)

- **类型**：`start`
- **功能**：接收用户输入 `sys.query`，作为整个流程的入口

### 2. 问题分类器 (Question Classifier)

- **类型**：`question-classifier`
- **模型**：GPT-5.5 (OpenAI)
- **温度**：0.7
- **分类**：

| 分类ID | 名称 | 说明 |
|--------|------|------|
| 1783925649868 | 产品信息 | 产品介绍、功能、型号、价格等 |
| 1783925681325 | 产品使用/售后 | 操作指导、故障排查、保修等 |
| 1783925698404 | 订单查询 | 物流、订单状态、支付问题等 |
| 1783925712643 | 退款问题 | 退款政策、申请流程、进度查询等 |

### 3. 知识检索节点 (Knowledge Retrieval)

4 个检索节点分别对应 4 个知识库：

| 节点 | 知识库 | 检索模式 | Top-K | Reranking |
|------|--------|----------|-------|-----------|
| 知识检索 1 | 产品信息库 | 混合检索 | 5 | 关闭（自定义权重：向量0.7+关键词0.3） |
| 知识检索 2 | 使用售后库 | 混合检索 | 4 | 关闭 |
| 知识检索 3 | 订单查询库 | 混合检索 | 4 | 关闭 |
| 知识检索 4 | 退款政策库 | 混合检索 | 4 | 关闭 |

**检索配置说明**：
- 知识检索 1 使用了自定义权重（向量 0.7 + 关键词 0.3），Embedding 模型为 `text-embedding-3-large`
- 其余节点使用默认 Reranking 模型配置

### 4. LLM 节点

4 个 LLM 节点使用相同模型但不同 Prompt：

- **模型**：GPT-5.4 (OpenAI)
- **Context**：各节点引用对应知识检索节点的 `result`

#### LLM 1 — 产品信息客服

```
角色：星眠 SmartSleep 官方客服助手
职责：产品介绍、功能说明、型号区别、价格、促销、规格参数、购买渠道
约束：仅基于知识库回答，不编造信息，不回答使用/售后/订单/退款问题
```

#### LLM 2 — 使用/售后客服

```
角色：星眠 SmartSleep 官方客服助手
职责：首次使用、App绑定、蓝牙连接、功能操作、清洁保养、故障排查、保修维修
约束：步骤用编号列出，安全隐患立即提醒停止使用，不回答产品价格/订单/退款问题
```

#### LLM 3 — 订单查询客服

```
角色：星眠 SmartSleep 官方客服助手
职责：订单状态、物流跟踪、修改地址、发票申请、支付异常
约束：需用户提供订单号/手机号/下单平台，退款问题引导至退款专席
```

#### LLM 4 — 退款问题客服

```
角色：星眠 SmartSleep 官方客服助手
职责：退款/退货政策、申请流程、到账时间、进度查询、退货寄回
约束：政策以知识库为准，情绪安抚优先，安全问题优先提醒
```

### 5. 直接回复节点 (Answer)

4 个回复节点分别输出对应 LLM 的 `text` 字段，无额外处理。

## 数据流

```
sys.query
    │
    ▼
问题分类器 ──┬── 产品信息 ──→ 知识检索1 ──→ LLM1 ──→ 回复1
             ├── 使用售后 ──→ 知识检索2 ──→ LLM2 ──→ 回复2
             ├── 订单查询 ──→ 知识检索3 ──→ LLM3 ──→ 回复3
             └── 退款问题 ──→ 知识检索4 ──→ LLM4 ──→ 回复4
```

## 设计决策

### 为什么用分类路由而不是单 LLM？

1. **精准检索**：每个分类只检索对应知识库，减少噪音，提高召回率
2. **专属 Prompt**：每个分支的 Prompt 针对性优化，回答更专业
3. **成本优化**：分类用轻量模型(GPT-5.5)，回答用更强模型(GPT-5.4)
4. **可维护性**：各分支独立，修改一个不影响其他

### 为什么知识检索 1 用自定义权重？

产品信息类问题中，关键词匹配（型号名如 SS-100、具体价格数字）的精确度更重要，因此调高了关键词权重至 0.3（默认通常更低）。

## 扩展建议

- 增加「人工转接」分支：当分类器置信度低时，直接转人工
- 增加对话记忆：当前为单轮问答，可加入 `conversation_variables` 实现多轮
- 增加敏感词过滤：启用 `sensitive_word_avoidance` 功能
- 增加图片理解：启用 Vision 功能，支持用户上传产品照片
