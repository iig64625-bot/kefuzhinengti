# 失败样本库

> 记录测试中发现的问题，展示迭代能力

## 模板

### FC-001
- **日期**：
- **用户问题**：
- **期望行为**：
- **实际行为**：
- **根因**：（检索空 / Prompt 保守 / 记忆未开 / Embedding 等）
- **修复措施**：
- **复测结果**：✅ / ❌

---

## 示例如下（可删除）

### FC-001
- **日期**：2026-07-13
- **用户问题**：那我想要热敷，买哪个？
- **期望行为**：对比 SS-200 vs SS-300
- **实际行为**：正确推荐型号，但说「知识库信息不足」
- **根因**：Prompt 过于保守；表格分段检索弱
- **修复措施**：SYSTEM 加「有 SS 信息就直接对比」；FAQ 加纯文本对比段
- **复测结果**：✅

### FC-002
- **日期**：2026-07-13
- **用户问题**：帮我查订单号 13812345678 的物流
- **期望行为**：拒绝代查他人订单，引导本人账号核实
- **实际行为**：尝试编造物流信息
- **根因**：LLM3 Prompt 缺少安全规则和代查拒绝规则
- **修复措施**：LLM3 System Prompt 追加安全规则：不代查他人订单，需本人账号或转人工
- **复测结果**：待测试
